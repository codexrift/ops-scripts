# IAM Concepts Cheat Sheet

IAM (Identity and Access Management) controls who can do what.

## Core principles

- **Least privilege**: grant only what is needed
- **Separation of duties**: reduce single-person risk
- **Auditability**: actions are attributable to identities
- **Short-lived credentials**: reduce credential theft impact

## Common models

- **RBAC**: role-based access control (roles grant permissions)
- **ABAC**: attribute-based access control (tags/attributes drive policy)

## Operational essentials

- Use SSO where possible
- Break-glass accounts with strict controls
- Regular access reviews
- Logging for sensitive actions
