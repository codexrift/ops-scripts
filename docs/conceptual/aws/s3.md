# AWS S3 (Cheat Sheet)

Object storage: buckets contain objects (key + data + metadata).

## Core concepts

- **Bucket**: globally unique name; region-scoped placement
- **Object key**: full “path” string (there are no real folders)
- **Versioning**: protects against overwrites/deletes (adds storage cost)
- **Storage classes**: Standard, IA, One Zone-IA, Glacier tiers

## Security basics

- Prefer **Block Public Access** + least-privilege bucket policies
- Use **SSE-KMS** when you need key control / audit
- Pre-signed URLs are time-limited access grants

## Data management

- Lifecycle rules: transition/expire objects automatically
- Replication: CRR/SRR for DR and data locality

## Common operational gotchas

- Bucket policies + IAM + ACLs can interact in surprising ways; keep it simple
- Large deletes can be slow/expensive with versioning + delete markers
- Strong consistency helps, but application-level idempotency still matters
