| Intent (Knowledge Type) | Core Question                                    | Description                                                     | Typical Artifacts                             | Standard Source                           |
|-------------------------|--------------------------------------------------|-----------------------------------------------------------------|-----------------------------------------------|-------------------------------------------|
| Conceptual              | What is it? How does it work? Why does it exist? | Explains models, principles, and context to build understanding | System overviews, workflows, explanatory docs | Diátaxis (Explanation)                    |
| Procedural              | How do I do it?                                  | Step-by-step instructions to perform a task                     | How-to guides, runbooks, setup instructions   | Diátaxis (How-to Guides)                  |
| Reference               | What are the exact commands, values, or specs?   | Structured lookup information with no narrative or workflow     | Cheat sheets, API docs, configuration tables  | Diátaxis (Reference)                      |
| Diagnostic              | What’s wrong? How do I fix it?                   | Helps identify and resolve issues efficiently                   | Troubleshooting guides, known error databases | Google SRE, PagerDuty (Incident Response) |
| Operational             | What do I do regularly to keep things running?   | Defines recurring tasks and operational routines                | Maintenance runbooks, monitoring procedures   | Google SRE (Runbooks)                     |
| Architectural           | How is this system designed and structured?      | Describes system components, interactions, and design decisions | Architecture diagrams, design documents, ADRs | ISO/IEC/IEEE 42010                        |
| Policy                  | What rules or standards must be followed?        | Defines constraints, compliance, and best practices             | Policies, standards, security guidelines      | ISO (e.g., ISO 27001)                     |
